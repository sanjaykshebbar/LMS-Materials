# Foundations of IT Support

Foundations of IT Support is a beginner-friendly course that teaches the core skills needed to start a career in helpdesk or desktop support. It covers computer hardware, operating systems, basic networking, troubleshooting, software support, and customer service so learners can confidently handle everyday technical issues.

The course is designed to help you understand how IT support works in real workplaces. You will learn how to diagnose common problems, assist users clearly, document issues properly, and build the practical foundation needed for entry-level IT roles.

## Role and responsibilities of IT Support (L1/L2/L3 tiers, shift models)

This module explains how IT support works at different levels and during different shifts. It covers L1, L2, and L3 responsibilities, ticket handling, escalation, and handovers so learners understand how support teams keep systems and users running smoothly.

# 🎓 IT Support Fundamentals

## Chapter 1 — Introduction to Enterprise IT Support

> Audience: Beginners with no IT background

---

# 🌍 What is IT Support?

IT Support ensures employees can use technology efficiently, securely, and with minimal downtime.

Technology includes:

- 💻 Laptops and Desktops
- 📧 Email
- ☁️ Microsoft 365 / Google Workspace
- 🌐 Wi-Fi and Internet
- 🔐 VPN and Identity
- 🛡️ Security Tools
- 📹 Meeting Rooms
- 🖨️ Printers
- 📱 Mobile Devices
- 📦 Business Applications

---

# Why IT Support Exists

Business depends on technology.

If systems fail:

- Employees cannot work
- Customers are affected
- Revenue may be lost
- Security risks increase

IT Support restores services quickly while protecting business operations.

# 🎯 Learning Objectives

By the end of this lesson, you should be able to:

- Explain the role of an L1 Support Engineer.
- Handle incidents and service requests.
- Communicate professionally with users.
- Perform structured troubleshooting.
- Document tickets clearly.
- Understand when and how to escalate to L2.

---

# 1. What is an L1 Support Engineer?

An **L1 (Level 1) Support Engineer** is the first technical person a user interacts with after reporting an IT issue.

L1 engineers are the face of the IT department. They receive requests, gather information, perform initial troubleshooting, resolve common issues, and escalate complex problems.

## Primary Goals

- Restore user productivity quickly.
- Resolve common issues during the first interaction.
- Record accurate ticket information.
- Maintain excellent customer service.
- Escalate only when necessary.

---

# 2. Where Does L1 Fit?

```text
Employee
   │
   ▼
Service Desk / L1
   │
   ▼
L2 Support
   │
   ▼
L3 Support
   │
   ▼
Vendor / Engineering
```

L1 owns the ticket from the moment it is created until it is either resolved or handed over with complete documentation.

---

# 3. A Typical Day

## Start of Shift

- Review open tickets
- Check overnight incidents
- Read shift handover notes
- Review monitoring alerts
- Check email and collaboration tools

## During the Day

- Answer calls
- Respond to chats
- Resolve tickets
- Configure devices
- Reset passwords
- Install approved software
- Follow up on pending requests
- Update documentation

## End of Shift

- Update all tickets
- Hand over unresolved work
- Notify next shift of critical issues

---

# 4. Types of Work

## Incidents

An unplanned interruption.

Examples:

- Outlook not opening
- Laptop won't boot
- VPN disconnected
- Teams camera not working

## Service Requests

Standard requests.

Examples:

- Software installation
- New monitor
- Access request
- Password reset
- New employee laptop

---

# 5. Ticket Lifecycle

```text
Issue Reported
      │
Ticket Created
      │
Assigned to L1
      │
Investigate
      │
Resolved?
 ├── Yes → Verify → Close
 └── No  → Escalate to L2
```

Every action should be documented.

---

# 6. Professional Troubleshooting Method

1. Understand the issue.
2. Ask relevant questions.
3. Reproduce the problem if possible.
4. Check for recent changes.
5. Perform least-risk troubleshooting.
6. Verify the solution.
7. Document everything.
8. Close or escalate.

---

# 7. Common Responsibilities

## Identity & Access

- Password resets
- Unlock accounts
- MFA assistance
- Verify identity

## Hardware

- Laptop setup
- Monitor issues
- Docking station
- Keyboard and mouse
- Webcam
- Headset

## Software

- Microsoft 365
- Adobe Reader
- Browsers
- Approved enterprise software

## Networking

- Wi-Fi
- VPN
- Ethernet
- Basic DNS/IP checks

## Meeting Rooms

- Teams Room devices
- Cameras
- Microphones
- Displays

---

# 8. User Communication

A good L1 engineer communicates clearly.

## Good Practices

- Introduce yourself.
- Listen without interrupting.
- Ask clear questions.
- Avoid unnecessary technical jargon.
- Explain what you are doing.
- Confirm the issue is resolved.

Example:

> "I have reset your VPN profile. Could you please try connecting again while I stay on the call?"

---

# 9. Documentation

A professional ticket should include:

- Problem summary
- Business impact
- Troubleshooting steps
- Findings
- Resolution
- Attachments
- Next action

## Good Example

```text
User unable to connect to VPN.

Verified identity.
Internet working.
VPN profile recreated.
Issue resolved after reconnect.
User confirmed.
```

---

# 10. Escalation

Escalate when:

- No documented solution exists.
- Administrative privileges are required.
- Infrastructure changes are needed.
- Multiple users are affected.
- SLA is at risk.
- Vendor support is required.

Always include:

- Problem description
- Business impact
- Logs
- Error messages
- Steps already attempted

---

# 11. Common Enterprise Tools

| Category | Examples |
|-----------|----------|
| ITSM | ServiceNow, Freshservice, Jira Service Management, ManageEngine |
| Identity | Active Directory, Microsoft Entra ID |
| Endpoint | Microsoft Intune, SCCM, JAMF |
| Remote Support | BeyondTrust, TeamViewer, AnyDesk |
| Collaboration | Microsoft Teams |

---

# 12. Key Performance Indicators (KPIs)

- First Contact Resolution (FCR)
- SLA Compliance
- Customer Satisfaction (CSAT)
- Average Resolution Time
- Ticket Backlog
- Documentation Quality

---

# 13. Common Mistakes

- Closing tickets without confirmation.
- Poor documentation.
- Escalating too early.
- Ignoring SLA deadlines.
- Making unapproved changes.
- Assuming instead of verifying.

---

# 14. Real-World Scenario

## User Complaint

"My Outlook is not opening."

### Questions

- When did it start?
- Any error message?
- Does Outlook open in Safe Mode?
- Any recent updates?

### Troubleshooting

- Restart computer
- Open Outlook in Safe Mode
- Disable add-ins
- Create new Outlook profile
- Run Office Repair

If unresolved, collect logs and escalate to L2.

---

# 15. Hands-on Practice

Practice these tasks:

- Reset a password
- Unlock an account
- Install software
- Configure Outlook
- Connect to VPN
- Troubleshoot printer
- Create a professional ticket
- Escalate a ticket correctly

---

# 16. Interview Preparation

## Sample Questions

1. What is the role of an L1 Engineer?
2. Difference between an Incident and a Service Request?
3. What is SLA?
4. What is First Contact Resolution?
5. Explain your troubleshooting methodology.
6. When should you escalate to L2?
7. How do you handle an angry user?

---

# 17. Summary

An L1 Support Engineer is responsible for restoring user productivity through first-line support.

The role combines:

- Technical troubleshooting
- Customer service
- Documentation
- Ticket ownership
- Communication
- Time management
- Escalation

A successful L1 engineer resolves common issues efficiently while ensuring every interaction is professional, well-documented, and aligned with organizational processes.
